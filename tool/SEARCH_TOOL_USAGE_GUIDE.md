# 搜索工具使用指南 (Search Tool Usage Guide)

## 概述

本工具是一个多模式搜索辅助系统，通过4个专用模式在对话的不同阶段提供支持。**核心原则：工具应在每个相关提示词上被调用，而不是只在对话开头调用一次。**

---

## 四大模式

### 1. [analyze-mode] — 项目理解
- **用途：** 在对话开始时分析项目、主题或研究领域
- **使用时机：** 收到新任务时的第一次调用
- **输出：** 项目概览、关键主题识别、搜索策略建议

**示例调用：**
```
[analyze-mode: 用户询问"如何撰写高质量的学术论文搜索查询"]
```

### 2. [search-mode] — 搜索策略讨论
- **用途：** 讨论搜索查询写法、搜索策略、数据库选择
- **使用时机：** 当对话进入"如何搜索"的讨论阶段
- **输出：** 查询构造技巧、布尔逻辑建议、数据库推荐

**示例调用：**
```
[search-mode: 讨论PubMed vs Google Scholar的搜索策略差异]
```

### 3. [assemble-mode] — 响应组装
- **用途：** 在最终答案呈现前，组装各部分信息
- **使用时机：** 收集到足够信息，准备构建完整响应时
- **输出：** 结构化响应框架、信息整合、逻辑验证

**示例调用：**
```
[assemble-mode: 组装关于搜索查询撰写的详细指导]
```

### 4. [final-standards] — 质量检查
- **用途：** 在最终答案呈现前，检查是否符合质量标准
- **使用时机：** 响应组装完成后、呈现给用户前
- **输出：** 质量评分、改进建议、标准合规性检查

**示例调用：**
```
[final-standards: 检查最终响应是否符合指导标准]
```

## 工作流示例

### 标准工作流（4步完整流程）

```
用户: "如何写好搜索查询？"

→ Step 1: [analyze-mode: 分析用户需求和背景]
→ Step 2: [search-mode: 讨论查询构造策略]
→ Step 3: [assemble-mode: 组装完整指导]
→ Step 4: [final-standards: 质量检查]
→ 最终回答呈现给用户
```

### 简化工作流（当场景不需要全部4步时）

```
用户: "PubMed怎么用？"

→ Step 1: [analyze-mode: 分析具体工具使用问题]
→ Step 2: [search-mode: 针对PubMed的搜索技巧]
→ 直接回答（无需assemble和final-standards）
```

## 关键使用原则

### ✅ 正确做法

1. **每个相关提示词都调用工具**
   - 对话演进时，模式应同步推进
   - 不要只在对话开头调用一次就结束

2. **根据对话阶段选择模式**
   - 讨论搜索策略 → [search-mode]
   - 组装最终答案 → [assemble-mode]
   - 质量把关 → [final-standards]

3. **模式可以组合使用**
   - 复杂任务可能需要多次调用同一模式
   - 可以在[search-mode]和[assemble-mode]之间来回切换

### ❌ 错误做法

1. ❌ 只在对话开头调用一次[analyze-mode]，之后再也不调用
2. ❌ 在讨论搜索策略时不用[search-mode]
3. ❌ 在呈现最终答案前不调用[final-standards]
4. ❌ 跳过[assemble-mode]直接呈现未组装的信息

## 模式触发关键词

| 触发词/场景 | 推荐模式 |
|---|---|
| 新任务、新主题、新项目 | [analyze-mode] |
| 搜索查询、搜索策略、数据库选择 | [search-mode] |
| 组装、整合、构建响应 | [assemble-mode] |
| 最终答案前、质量检查、标准合规 | [final-standards] |
| "如何写搜索查询" | [search-mode] → [assemble-mode] |
| "帮我搜索XX" | [analyze-mode] → [search-mode] |
| "总结一下" | [assemble-mode] → [final-standards] |

## 常见问题

**Q: 每次对话都需要调用所有4个模式吗？**
A: 不一定。根据对话复杂度和阶段，可以只调用相关模式。但**核心原则是：每个相关提示词都应调用工具**。

**Q: 可以在同一轮对话中多次调用同一模式吗？**
A: 可以。复杂任务可能需要反复讨论搜索策略（多次[search-mode]）或多次组装（多次[assemble-mode]）。

**Q: 如果用户直接问"帮我搜索XX"，应该用哪个模式？**
A: 先用[analyze-mode]理解搜索需求，再用[search-mode]制定搜索策略，然后执行搜索，最后用[assemble-mode]和[final-standards]组装和检查结果。
